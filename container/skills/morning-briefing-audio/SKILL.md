# Morning Briefing Audio

Turn `/workspace/group/briefings/YYYY-MM-DD.md` into a single narrated MP3 plus a `.txt` transcript under `/workspace/group/audio/`. The publish step (Task 6 in the spec) is a **separate** scheduled task at 06:20 SAST — this skill stops once the concatenated MP3 + transcript are on disk.

## Why 3 segments + concat

OpenAI's `gpt-4o-mini-tts` accepts up to **4096 characters per call**. A typical daily briefing exceeds that (~4,250 chars). Rather than splitting on arbitrary paragraph boundaries, split on the briefing's natural section boundaries — that way each segment is comfortably under the limit, segment audio matches segment content (helps debugging), and the segments can be regenerated individually if one comes out wrong.

The 3 segments map cleanly to the markdown:

| # | Segment       | Markdown sections                                | Intro / outro                                                      |
|---|---------------|--------------------------------------------------|--------------------------------------------------------------------|
| 1 | International | `## Weather — Johannesburg` + `## International` | **Intro:** `Good morning, Peter. This is your briefing for {Weekday}, {Day} {Month} {Year}.` |
| 2 | South Africa  | `## South Africa (TimesLive)`                    | (no intro/outro — straight into content)                           |
| 3 | Tasks         | `## Your Tasks for Today`                        | **Outro:** `That is the end of your briefing. Have a good day.`    |

Concat order: 01-international → 02-sa → 03-tasks. The final concatenated MP3 sounds like one continuous broadcast.

## Inputs / outputs

- **In:** `/workspace/group/briefings/YYYY-MM-DD.md` (today's date in SAST = Africa/Johannesburg = UTC+2)
- **Out:**
  - `/workspace/group/audio/YYYY-MM-DD-01-international.mp3` — intermediate segment 1
  - `/workspace/group/audio/YYYY-MM-DD-02-sa.mp3` — intermediate segment 2
  - `/workspace/group/audio/YYYY-MM-DD-03-tasks.mp3` — intermediate segment 3
  - `/workspace/group/audio/YYYY-MM-DD.mp3` — **final concatenated episode** (this is what publish uploads)
  - `/workspace/group/audio/YYYY-MM-DD.txt` — full narration script across all 3 segments (= TMQ transcript)

If the input markdown is missing, log `briefing audio: source markdown not found, skipping` and exit cleanly. **Do not retry** — there's nothing to convert.

Create `/workspace/group/audio/` if it doesn't exist.

## Step 1 — Split the markdown into 3 segments

Read the markdown. Identify the four section boundaries by their headings (`## Weather — Johannesburg`, `## International`, `## South Africa`, `## Your Tasks for Today`). Build three segment strings:

- **Segment 1 (International):** weather block + international stories
- **Segment 2 (South Africa):** SA stories
- **Segment 3 (Tasks):** today's + overdue tasks (or the "No tasks due today." line)

Apply the cleaning pipeline below to each segment **independently**. Then prepend the intro line to segment 1 and append the outro line to segment 3.

### Cleaning pipeline (apply to each segment, in order)

1. **Strip the report-generator footer** if present (e.g. `_Report generated by NanoClaw_`).
2. **Drop the top `# Morning Briefing — ...` heading** if it's in this segment.
3. **Replace section headings** with spoken lead-ins (mind the blank lines — they become natural pauses):
   - `## Weather — Johannesburg` → `\n\nWeather for Johannesburg.\n\n`
   - `## International` → `\n\nInternational news.\n\n`
   - `## South Africa (TimesLive)` → `\n\nSouth Africa headlines.\n\n`
   - `## Your Tasks for Today` → `\n\nYour tasks for today.\n\n`
4. **Drop horizontal rules** (lines that are just `---`).
5. **Convert markdown links** `[text](url)` → just `text`.
6. **Drop URL-only lines** (a line whose only content is `http…` or `(http…)`). URLs do not narrate well.
7. **Strip code blocks** (everything between triple backticks). If any existed, replace with the sentence: `Code block omitted from audio.`
8. **Strip tables** (any block of lines beginning with `|`). If any existed, replace with: `A table is shown in the written report.`
9. **Strip remaining markdown formatting characters**: `*`, `_`, `` ` ``, `~`, `#`.
10. **Convert bullet lines** to prose: at line start, strip `- `, `* `, `• `, and ensure each bullet ends with `.` (full stop).
11. **Normalise abbreviations and units** using this dictionary (extend over time as you spot mispronunciations):
    - `R ` immediately followed by a number → `<number> rand`
    - `USD/ZAR` → `the dollar to rand exchange rate`
    - `SARB` → `S A R B` (letter-by-letter)
    - `VFS` → `V F S`
    - ` SA ` standalone → ` South Africa `
    - `°C` → ` degrees Celsius`
    - `km/h` → ` kilometres per hour`
    - `%` → ` percent`
    - `AI` → `A I`
    - `xAI` → `x A I`
    - `OpenAI` → `Open A I`
    - `SpaceX` → `Space X`
    - `URL` → `U R L`
12. **Collapse runs of 3+ blank lines** down to two.
13. **Segment 1 only:** prepend `Good morning, Peter. This is your briefing for {Weekday}, {Day} {Month} {Year}.\n\n`
14. **Segment 3 only:** append `\n\nThat is the end of your briefing. Have a good day.`

After cleaning, **assert each segment is ≤ 4096 chars**. If any exceed it (e.g. an unusually heavy news day), the over-segment must be split further on paragraph boundaries before TTS. Log a warning when this happens.

Concatenate segment-1 + `\n\n` + segment-2 + `\n\n` + segment-3 and write the joined script to `/workspace/group/audio/YYYY-MM-DD.txt` — this is the TMQ transcript (one file representing the whole episode).

## Step 2 — Call OpenAI TTS once per segment

**Auth via OneCLI inject.** OpenAI is registered as a OneCLI secret with host pattern `api.openai.com` and inject `Authorization: Bearer {value}`. The HTTPS proxy adds the real header in flight. **Do NOT send your own `Authorization` header** — if you do, your placeholder reaches the server unmodified and OpenAI returns `401`.

Call the TTS endpoint three times — once per segment — with the **same `voice` and the same `instructions` every call**. Consistency is the whole point of the steerable model.

```
POST https://api.openai.com/v1/audio/speech
Headers:
  Content-Type: application/json
  # NO Authorization header — OneCLI injects it
Body (JSON):
{
  "model": "gpt-4o-mini-tts",
  "voice": "sage",
  "input": "<segment text>",
  "instructions": "<news-anchor block, verbatim, see below>",
  "response_format": "mp3",
  "speed": 1.0
}
```

Stream each response to its segment file:
- segment 1 → `YYYY-MM-DD-01-international.mp3`
- segment 2 → `YYYY-MM-DD-02-sa.mp3`
- segment 3 → `YYYY-MM-DD-03-tasks.mp3`

### News-anchor instructions block — use verbatim, every segment

The `instructions` parameter is the steerable "vibe" exposed as "Vibe" on openai.fm. **Do not paraphrase or shorten** between segments — drift here causes audible tonal shifts across the catalogue.

```
Voice Affect: Calm, composed, professional. Project quiet authority without sounding stern.

Tone: Like a morning news anchor on a quality public broadcaster. Measured. Trustworthy. Not theatrical.

Pacing: Steady and unhurried. Slight natural pauses between stories and at section transitions. Do not rush.

Emotion: Neutral and informative. Slightly warmer for opening greetings and closings. Matter-of-fact for news content. Light touch only — no dramatic emphasis.

Pronunciation: Clear articulation. Treat acronyms read as letters as letter-by-letter (S A R B, V F S). Pronounce place names carefully.
```

## Step 3 — Concatenate with ffmpeg

`ffmpeg` is installed in the briefing container's overlay image. Use the concat **demuxer** with `-c copy` — all three segments come from the same TTS endpoint with identical codec params, so this is lossless and fast (no re-encoding).

Write a concat list to a temp file, then run ffmpeg:

```bash
cat > /tmp/concat-list.txt <<EOF
file '/workspace/group/audio/YYYY-MM-DD-01-international.mp3'
file '/workspace/group/audio/YYYY-MM-DD-02-sa.mp3'
file '/workspace/group/audio/YYYY-MM-DD-03-tasks.mp3'
EOF

ffmpeg -y -hide_banner -loglevel error \
  -f concat -safe 0 -i /tmp/concat-list.txt \
  -c copy \
  /workspace/group/audio/YYYY-MM-DD.mp3
```

If ffmpeg returns non-zero, log the stderr and **do not delete the segment files** — they're the recovery path. The publish task will see no final MP3 and skip.

## Step 4 — Capture duration with ffprobe

```bash
ffprobe -v error -show_entries format=duration \
        -of default=noprint_wrappers=1:nokey=1 \
        /workspace/group/audio/YYYY-MM-DD.mp3
```

Round to the nearest second. Save it somewhere the publish task can read (e.g. `audio/YYYY-MM-DD.duration` — a single integer file). If ffprobe fails for any reason, log and continue — publish will send without `audio_duration` and Apple Podcasts will show duration after first play.

## Step 5 — Log and finish

Log one line: date, per-segment char counts, per-segment MP3 sizes, final MP3 size, final duration in seconds, total time taken in ms. Do **not** chat-message anyone — this task is non-interactive.

## Failure handling

- **Markdown file missing:** log `briefing audio: source markdown not found, skipping` and exit cleanly. No retry.
- **A single segment cleans to over 4096 chars after split:** further-split that segment on paragraph boundaries before TTS. Log a warning.
- **OpenAI returns 4xx on any segment:** log status + body, and `do not retry`. If it's `401`, double-check you did NOT send your own `Authorization` header — the OneCLI inject covers `api.openai.com`. If still `401`, the OneCLI secret is missing or its host pattern doesn't match.
- **OpenAI returns 5xx or network error on any segment:** retry that segment once after 60 seconds. If second attempt fails, log final failure and exit — partial output stays on disk for inspection but no final MP3 is built.
- **ffmpeg concat fails:** log stderr and exit. Keep segments on disk.
- **Disk write fails:** log and exit. Nothing useful to retry.

## Publish (separate task at 06:20 SAST)

Once this skill exits cleanly, the publish task takes over. It:
- Reads `/workspace/group/audio/YYYY-MM-DD.mp3`, `.txt`, `.duration`
- POSTs `multipart/form-data` to `https://task-master-quest-production.up.railway.app/api/briefings`
  with fields `feed_slug=morning-briefing`, `date=YYYY-MM-DD`, computed `title`, `transcript`, `audio_duration`, and `audio` file upload
- Sends **no** Authorization header — OneCLI proxy injects it for the TMQ host
- Treats HTTP 200 as success, logs the returned `audio_url`. 401/4xx don't retry; 5xx/network retries with exponential backoff (60s, 180s, 600s)


Not part of this skill — add when the catalogue is established.
